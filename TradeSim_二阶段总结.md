# TradeSim 智能回测系统 - 二阶段总结与技术迭代透析

> 本文档对 TradeSim 系统的第二阶段（UI/UX核心痛点解决与 AI 研判大脑接入）进行全景回顾。一阶段我们构建了硬核引擎，二阶段我们赋予了系统更好的交互性与不可思议的“分析灵魂”。

---

## 🎯 一、 业务层面体验升级 (UX & Product Enhancement)

### 1. 治愈配置焦虑，渐进式表单设计
*   **消灭 UI 臃肿**：引入“渐进式表单设计 (Progressive Disclosure)”，将佣金滑点等专业阻碍感较强的参数封存在高亮折叠面板中，保持主界面的极简与清爽。
*   **网格间距“双模支持与即时换算”**：新增了对网格切分的“等比（百分比）”与“等差（固定价差）”双模式支持。并在下方实时提示“预估切割格数”与“预估单格极差”，极大缓解了用户的算术焦虑与心智负担。

### 2. 回放体验升级与图表微观重塑
*   **日期区间组件集成**：砍掉了双拼的独立日期框，采用一站到底的 Element Plus `daterange` 日期选择器，彻底解决了排版不均与文字挤压。
*   **四轴宏大图表连动**：为了承载更多的因子表达，图表由单图垂直延展成超大的“四轴微观连动图”（真实股价与网格线 + 策略与标的收益对跑 + 动态仓位利用率 + 痛苦回撤线）。
*   **空间防撞击与刻度留白**：通过在 ECharts 底层调整 grid 分配与 Y 轴 `nameGap` 属性，彻底消灭了图名字母、网格横线与数字刻度重叠在一起的视觉污染。

### 3. 数据维度的时空下潜
*   **突破“日线”诅咒**：一阶段系统只能进行粗粝的日级别复盘。二阶段彻底打通了 AkShare 与后端引擎的五分钟、一分钟级的高频切片通道。用户可切换“极致穿透”环境，捕捉日内高频网格波段。

## 🧠 二、 "量化AI大脑" - 架构与通信深度重构 (AI & Architecture)

二阶段最重量级的里程碑，在于引入了 Qwen（千问）等先进大语言模型，结合提示词工程（Prompt Engineering）彻底重置了交互范式，打造了一个“随叫随到、冰冷毒舌”的量化医生。

### 1. 流式推理网关 (`Server-Sent Events`)
*   传统的 `HTTP` 请求对于大模型而言等候异常漫长。我们启用了 FastAPI 的 `StreamingResponse` 事件流机制。
*   每次用户召唤 AI 时，前端抽屉都会收到切片式的字符流灌注。Markdown 解析库会逐字排版、动态防抖，实现了如顶级商业闭源级别的丝滑打印体感。

### 2. Vue3 渲染保护壁垒 (Render Context Isolation)
*   **白屏熔断机制防护**：修复了因引入 `DOMPurify` 和 `marked` 并遗漏核心 API 绑定导致组件被强制卸载（导致全站白屏宕机Bug）。
*   加装了纯逻辑层的 `try-catch` 防御罩。即使网络阻断或大模型发癫吐出无法闭合的错乱语法，Vue 组件也能从容拦截并自动退化展示，绝不再伤及无辜。

### 3. "机构级内潜推理" - 提示词黑魔法 (Prompt Engineering)
我们彻底重构了后端代码深处的 Prompt 内核映射逻辑，将其由“散文小结”升级为具备严密归因闭环的“诊断器”：
*   **思维链强制锚点**：AI 在向用户吐字前，被隐性约束要求在心里推演并挑选主次核心指标（区间错配、极值崩溃、市场形态或磨损摩擦）。
*   这样彻底杜绝了模型算数差所带来的幻觉 (Hallucination)。让它把所有结论均强行挂靠在我们传递进去的参数和数值极点上。直截了当地点破回测跑出极端利润或惨重折损的物理学本源。

## 🛠️ 三、 历史重构与后向兼容 (Backward Compatibility)

*   **MySQL DDL 动态探针与防御补丁**：我们通过 Python 脚手架植入了 DDL 脚本，使用 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`，使得线上的主库能够毫发无伤地扩容新增的参数维度。
*   **前端双模混排**：面对 MongoDB 冷库中翻出来的旧版二维结构历史快照，前端代码成功实现了“基于属性有无探针”的分流。如果侦测到老年代数据，图表会平滑退回到双轴兼容模式。一处代码，安顿好了新老全部的数据展现。
