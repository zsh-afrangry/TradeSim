# TradeSim 开发文档

> 开发者视角：重新打开项目时需要知道的一切。

---

## 1. 技术栈

| 层 | 技术 |
|---|---|
| 后端框架 | FastAPI + Uvicorn（全异步） |
| 数据处理 | Polars + PyArrow（替代 Pandas） |
| 行情数据源 | AkShare |
| 关系型数据库 | MySQL（开发）/ SQLite（单机打包） |
| 文档型数据库 | MongoDB + Motor（开发）/ 本地 JSON 文件（单机打包） |
| ORM | SQLAlchemy |
| 前端框架 | Vue 3 + Vite + Element Plus + ECharts |
| AI 调用 | OpenAI 兼容接口（默认接阿里云 DashScope / Qwen） |
| 打包 | PyInstaller（单目录模式） |

---

## 2. 运行模式

通过 `backend/.env` 中的 `APP_MODE` 字段控制：

```dotenv
APP_MODE=development   # 联机开发（默认）
APP_MODE=standalone    # 单机打包交付
```

| 维度 | development | standalone |
|---|---|---|
| 关系型存储 | MySQL | SQLite（自动创建 `data/tradesim.db`） |
| 大对象存储 | MongoDB | 本地 JSON 文件（`data/logs/*.json`） |
| 前端服务 | Vite 独立服务器（端口 5173） | FastAPI 托管 `static/`（端口 8000） |
| motor / bson | 需要安装 | 不导入，避免打包体积膨胀 |

### .env 完整配置项

```dotenv
APP_MODE=development

# 关系型数据库（development 模式使用）
MYSQL_URL=mysql+pymysql://root:root@127.0.0.1:3306/tradesim

# MongoDB（development 模式使用）
MONGO_URL=mongodb://127.0.0.1:27017

# AI 大模型（两种模式均可用）
LLM_API_KEY=your_api_key_here
LLM_MODEL=qwen-max
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

## 3. 本地启动

```bash
# 后端（确保 MySQL 和 MongoDB 已启动）
cd backend
uvicorn main:app --reload

# 前端
cd frontend
npm run dev
```

后端默认监听 `127.0.0.1:8000`，前端 Vite 监听 `5173` 并代理 API 请求到后端。

## 4. 后端接口清单

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/simulate/run` | 执行回测，返回指标 + 资金曲线 + 交易流水 |
| POST | `/api/v1/records/save-favorite` | 收藏一次回测结果（写 MySQL + MongoDB） |
| GET | `/api/v1/records/list` | 获取收藏列表，按收益率倒序，最多 100 条 |
| GET | `/api/v1/records/detail/{id}` | 获取某条记录的完整详情（含曲线和流水） |
| POST | `/api/v1/ai/analyze-stream` | SSE 流式返回 AI 诊断报告 |

### POST /api/v1/simulate/run

请求体关键字段：

```json
{
  "symbol": "000400",
  "start_date": "2023-01-01",
  "end_date": "2024-01-01",
  "data_frequency": "daily",
  "initial_capital": 100000,
  "commission_rate": 0.00025,
  "slippage": 0.01,
  "strategy_name": "GRID_TRADING",
  "strategy_params": {
    "lower_bound": 20.0,
    "upper_bound": 30.0,
    "grid_type": "geometric",
    "grid_step_pct": 0.05,
    "grid_count": 20,
    "base_position_ratio": 0.5,
    "trade_mode": "amount",
    "funds_per_grid": 10000.0
  }
}
```

响应体：`{ metrics, equity_curve[], execution_records[] }`

## 5. 数据库表结构

### simulation_records（MySQL / SQLite）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK | 自增主键 |
| mongo_log_id | VARCHAR(50) | 关联 MongoDB / 本地 JSON 的凭证 ID |
| strategy_name | VARCHAR(50) | 策略标识，如 `GRID_TRADING` |
| symbol | VARCHAR(20) | 股票代码 |
| start_date / end_date | DATE | 回测区间 |
| data_frequency | VARCHAR(20) | 数据频率，默认 `daily` |
| strategy_params | JSON | 策略参数字典（可灵活扩展，无需改表结构） |
| total_return | DECIMAL(10,4) | 总收益率（建有索引，用于排行榜排序） |
| max_drawdown | DECIMAL(10,4) | 最大回撤率 |
| win_rate | DECIMAL(10,4) | 胜率 |
| total_trades | INT | 总交易笔数 |
| created_at | TIMESTAMP | 收藏时间 |

### 大对象存储（MongoDB collection: simulation_logs / 本地 JSON）

每条记录包含：
- `equity_curve`：资金曲线数组，每项含 `date, close_price, net_value, benchmark_value, position_utilization, drawdown`
- `execution_records`：交易流水数组，每项含 `timestamp, action, price, volume, amount, commission, slippage_cost`

两张"表"通过 `mongo_log_id` 关联，MySQL 存轻量索引，MongoDB/JSON 存大数组，互不拖累查询性能。

## 6. 策略层架构

```
backend/app/strategy/
├── base.py                  # BaseStrategy 抽象基类
└── specific/
    └── grid_trade.py        # GridTradingStrategy
```

**BaseStrategy** 提供：
- `_buy()` / `_sell()`：执行买卖并记录流水
- `_record_equity_snapshot()`：每个时间步记录净值快照
- `_calculate_metrics()`：计算收益率、回撤、胜率等指标
- 抽象方法 `execute()`：子类实现具体策略逻辑

**新增策略只需：**
1. 在 `specific/` 下新建类，继承 `BaseStrategy`，实现 `execute()`
2. 在 `backend/app/api/v1/simulate.py` 的路由中按 `strategy_name` 注册该类

**GridTradingStrategy 核心设计：**
每个网格价位是一个 `GridNode` 对象，状态为 `IDLE` 或 `OCCUPIED`。价格下穿格线时买入并将节点置为 `OCCUPIED`；价格上穿时卖出并 `release()` 节点。同一格位必须完成"一买一卖"配对后才能再次买入，防止资金在同一区间无限堆仓。

## 7. AI 调用

- 接口：`POST /api/v1/ai/analyze-stream`，返回 `text/event-stream`（SSE）
- 客户端：使用 `openai` Python SDK 的 `AsyncOpenAI`，`base_url` 指向 OpenAI 兼容接口
- 默认模型：阿里云 DashScope（`qwen-max`），可通过 `.env` 的 `LLM_BASE_URL` 换成任意兼容接口
- 前端通过 `EventSource` 或 `fetch` + `ReadableStream` 接收，逐字渲染 Markdown

---

## 8. 打包流程（build.bat）

在 `backend/` 目录下执行 `.\build.bat`，自动完成：

1. 编译前端 Vue 项目（`npm run build`）
2. 将 `frontend/dist/` 拷贝至 `backend/static/`
3. 在 `.env` 末尾临时追加 `APP_MODE=standalone`
4. 执行 PyInstaller 打包（`--onedir`，入口为 `run_app.py`）
5. 自动还原 `.env`（删除临时追加的行）

输出目录：`backend/dist/TradeSim回测引擎/`，双击 `TradeSim回测引擎.exe` 启动。

> 打包完成后 `.env` 自动还原，本地开发环境不受影响。
