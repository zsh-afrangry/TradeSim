# TradeSim 智能回测系统 - 阶段性全景总结与架构透析

> 本文档对当前已落地的 TradeSim 系统进行一次全面的盘点，分为**业务层面**与**技术层面**，为您接下来的深度分析和架构演进提供宏观视角的参考。

---

## 🎯 一、 业务层面总结 (Business Perspective)

### 1. 系统的核心价值定位
目前 TradeSim 已经脱离了单纯的代码脚本测试阶段，进化成了一个**高度产品化、开箱即用的量化交易回测实验室**。
它允许交易员或策略研究者脱离纯代码环境，通过直观的 Web UI 即可完成复杂策略的参数调优，并即时获取从资金曲线到逐笔买卖流水的全方位数据反馈。

### 2. 已落地并验证的核心业务流
*   **高仿真交易环境搭建**：支持对“起始时间”、“初始启动资金”、“双边佣金规费”以及“单边价格滑点”进行高精度配置，使回测环境尽可能逼近 A 股真实的挂单成交场景。
*   **首个交易策略 - 动态网格 (Grid Trading)**：
    *   成功实现了网格交易的闭环生命周期：首日根据 `底仓比` 自动计算并买入底仓。
    *   随后程序按照 `网格间距(%)` 生成价格刻度尺，只要日线收盘价跨越该阶梯，系统即自动根据设定的 `每格交易额` 触发高抛或低吸操作，以应对剧烈震荡市。
*   **可视化多维表现评估**：
    *   **核心四维金刚线**：直接输出【总收益率】、【期内最大回撤】、【总交割频次】和【估算胜率】，用最直观的数据衡量该参数组合的好坏。
    *   **资产穿透曲线 (Equity Curve)**：在时间轴上动态展现由行情波动造成的账面净值起伏，并阴影叠加同期的“回撤幅度”，让风险敞口一目了然。
*   **成果沉淀与复盘机制**：
    *   **量化组合排行榜**：所有的精彩回测不再随网页关闭而流失，点击一键“收藏”后，记录将直接存入系统。目前支持以收益率倒序生成排行榜单。
    *   **历史快照提档复原**：允许用户在未来的任意时刻调阅曾经的跑分档案（包含当时的入参快照、盈亏曲线及最后一百笔详细的手续费滑点流水）。

## 🛠️ 二、 技术层面总结 (Technical Perspective)

当前架构可以说是融合了多种前沿技术优势的**“性能极客”版**设计方案。

### 1. 前端架构工程 (Vue 3 + Vite + Element Plus)
*   **SPA 单路由架构**：采用 `Vue Router` 搭建了真正的前后解耦框架。用户在【实验室面板】和【历史收藏库】之间切换属于无感知的纯前端行为，体验极其流畅。
*   **大屏数据可视化**：深度整合 `ECharts` 引擎，利用前端 Canvas 高性能渲染由后端吐出的成百上千个交易日数据坐标点。
*   **人性化表单工程**：将专业且生涩的金融量化入参（如滑点、底仓规费等）利用 Element Plus 的双向绑定及 Hover Tooltip 技术进行了友好的图形化封装。

### 2. 后端服务端架构 (FastAPI + Python 3.10+)
*   **全异步高性能网关**：抛弃了传统的 Django/Flask 同步模型，应用全套 `async/await` 非阻塞机制处理网络请求（包含请求 AkShare 及向 MongoDB 写数据）。
*   **Pydantic 契约精神**：我们建立了一套极其完备的 Schema 体系。不仅前端输入受到严格安检，后端向数据库写入的大量嵌套 JSON 也因此获得了极其安全的类型约束。
*   **策略引擎解耦 (Strategy Pattern)**：
    *   系统定义了抽象的 `BaseStrategy`，它是整个量化世界的基石，封装了**“资金记录法则”、“胜率统计算法”与“生命周期抽象”**。
    *   具体的网格算法 `GridTradingStrategy` 单纯作为子类实现其 `execute()` 核心。这意味着未来如果我们想写一个“MACD 均线双叉策略”，只需新建一个数十行的类并注册到路由即可，核心的主循环和存库逻辑一行代码都不用改！

### 3. 数据处理引擎 (AkShare + Polars + PyArrow)
*   完全抽离了对速度较慢且内存利用率不佳的 Pandas 的依赖。
*   当从 AkShare 抓取到源外数据时，系统直接通过底层基于 C++ 实现的 `PyArrow` 零拷贝通道瞬间转置进入 `Polars`。
*   利用 Polars 的**多线程引擎和无情向量计算**，这为未来我们下探到“五分钟线”甚至“Tick 级高频极大数据量”测试提供了坚实的底层硬件支撑。

### 4. 👑 压轴核心架构：「温冷分离」异构集群双写模型
抛弃了将大长文章强行塞入关系型数据库的传统做法，我们在存证系统上采用了大厂极常用的极致解耦思路：
*   **热/温查询中枢 (MySQL - SQLAlchemy 驱动)**：用于放置并维护一个轻巧的主表 `simulation_records`。里面只存放需要做算术比对和索引排序的**标签、收益率、回撤及轻量级 JSON 配置**。它保障了前端请求列表页时能在几毫秒内闪电返回全量榜单。
*   **深海冷库沉淀区 (MongoDB - Motor 驱动)**：利用 NoSQL 专门拿来吃大体积 BSON 文档的天赋，将多达几兆字节的**资金走势大数组**和**流水长表**隔离存放。
*   **唯一羁绊**：通过给 MySQL 中颁发一个 `mongo_log_id`，使得跨数据库的关联查询成为可能，并且互不拖累性能。

## 🔮 三、 留给您的构想与延展空间 (Future Vision)

在此黄金地基之上，您可以从以下几个维度展望接下来的路线：

1.  **策略流派的扩张**：结合已经解耦的基类，您可以极其轻松地让 AI 再写诸如：双均线金叉死叉 (MACD)、海龟交易突破、甚至引入基于各种布林带 (BOLL) 方差的突破算法。
2.  **Benchmark (比较基准)**：在图表上再引入一根线——比如同时期的 `沪深300指数` 涨跌幅，让策略的资金曲线直接与其对标去计算 **Alpha (超额收益)** 和 **Beta (系统性风险)**。
3.  **大参数寻优引擎 (Grid Search Optimization)**：让程序自己暴力穷举 “网格间距 (0.01~0.1)” 与 “网格上下限”，去后台跑几万次回测，自动把最高收益率的那一组参数挑选出来推荐给您。
4.  **云端定时任务实盘拉响**：结合现在的回测结果，为系统加装 `APScheduler` 等定时器模块，每天挂载行情判断当前真实股价是否到达您所设置的网格线，通过邮件/企业微信提醒您实际操作！
